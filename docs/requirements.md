# 需求文档

## 1. 产品定位

**族谱系统（Family Tree）** 面向个人或单一家族，提供人物档案管理、亲属关系维护、族谱可视化浏览，以及面向族人/访客的公开展示（Portal）。

典型场景：管理员维护一个大家族（可达数十代），成员与游客通过展示端只读浏览；支持地理标记（族群分布、坟地）与人物检索。

## 2. 用户与角色

| 角色 | 码值 | 说明 |
|------|------|------|
| 游客 | — | 免登录访问 Portal 与 `/api/public/*` |
| 普通族民 | `member` | 注册默认角色；可登录展示端 |
| 信息发布员 | `editor` | 含 member；可维护地理标记等（按接口权限） |
| 管理员 | `admin` | 含 editor；管理后台 CRUD、用户角色 |

权限为**层级包含**（admin > editor > member）。管理后台路由需 `admin`。家族归属以 `families.owner_id` 为准。

## 3. 功能需求

### 3.1 账户与权限

- 用户注册、登录（JWT Bearer）
- 登录后按角色跳转：管理员进后台，其他人进 Portal
- 管理员可查看用户列表并修改角色（禁止取消最后一个 admin）

### 3.2 家族管理

- 创建 / 查询 / 更新 / 删除本账号所属家族
- 字段：名称、简介、籍贯（origin_place）
- 统计：人数、男女数、世代跨度

### 3.3 人物管理

- 人物 CRUD（姓名、小名、性别、辈分、出生年份等必填项）
- 可选：出生/去世日期、籍贯、电话、现住址、简介、备注、是否在世、头像
- 按关键词、辈分筛选与分页
- 附件上传与头像设置

### 3.4 亲属关系

- 库中仅存两类关系：`parent`（父母→子女）、`spouse`（配偶）
- 兄弟姐妹由「共同父亲/母亲」推导，不单独落库
- 支持创建与删除关系

### 3.5 族谱可视化

| 模式 | 说明 |
|------|------|
| 世系全图 | 全部成员按世代树形展开，可检索高亮 |
| 男系世系 | 从指定祖先向下男系主脉 |
| 以人为中心 | 指定人物上下查 N 世；向上 0 / 向下 0 仅本人 |

展示端与管理端共用 SVG 树形布局；支持缩放、拖拽、聚焦高亮。

### 3.6 公开展示端（Portal）

- 路径前缀：`/portal`
- 页面：家族首页、族谱、人物检索、人物档案、地图
- 游客可读；配置项可固定展示家族 ID

### 3.7 地理标记

- 类型：`distribution`（族群分布）、`cemetery`（坟地）
- 管理端手工录入名称、经纬度、地址、备注、可选关联人物
- 展示端高德地图展示；列表点击可定位跳转
- 需配置高德 Web 端（JS API）Key 与安全密钥

### 3.8 导入导出

- Excel 导入/导出人物与关系

## 4. 非功能需求

| 项 | 要求 |
|----|------|
| 响应格式 | 统一 `{ code, message, data }`；业务错误通常 HTTP 200 + `code != 0` |
| 浏览器 | 现代桌面浏览器（Chrome / Edge / Firefox 近两年版本） |
| 数据量 | 支持单家族数十代、数百至千级人物（全图谱慎用于超大图谱） |
| 安全 | JWT；密钥与数据库账号不入库；上传目录隔离 |

## 5. 范围外（当前版本不做）

- 多家族成员协作表（`family_members`）
- 移动端原生 App
- 地图地址自动地理编码（需手工经纬度）
- 复杂宗亲会多租户 SaaS 计费
